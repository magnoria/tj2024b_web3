

[할 일 관리 시스템 - REST API 구현 지문]
-목적
할 일(Todo) 관리 시스템을 구현하고, MVC2 3tire 계층 분리 및 엔티티 설계, DTO 사용, 공통 시간 정보(BaseTime) 처리까지 연습한다.

-기능 명세
1. 할 일 등록 기능
사용자는 새로운 할 일 정보를 등록할 수 있어야 한다.
등록 시 필요한 정보: 제목(title), 내용(content), 완료 여부(done)
등록 시간(createdDate)과 수정 시간(modifiedDate)은 자동으로 기록되어야 한다.

2.전체 할 일 조회 기능
저장된 모든 할 일 목록을 조회할 수 있어야 한다.

3.개별 할 일 조회 기능
특정 할 일의 상세 정보를 id를 통해 조회할 수 있어야 한다.

4.할 일 수정 기능
기존에 등록된 할 일의 정보를 수정할 수 있어야 한다.
수정 시 할 일의 id를 기준으로 변경하며, 수정 시간은 자동으로 갱신되어야 한다.

5.할 일 삭제 기능
할 일의 id를 기준으로 해당 할 일을 삭제할 수 있어야 한다.

6.페이징 기능이 적용된 할 일 목록 조회
할 일 목록을 페이지 단위로 조회할 수 있어야 하며, 페이지 번호(page)와 페이지당 항목 수(size)를 으로 입력받아 처리한다.
